Data Collection Disclosure for Re-Source

Last Updated: December 16, 2024

Purpose of This Document: This disclosure outlines all user data collected, shared, or transmitted by the Re-Source app, including data handled by third-party SDKs and libraries. This document is designed to provide complete transparency about our data practices and to fulfill Google Play Store Data Safety requirements.

Summary Overview

Data Category Collected Shared with Third Parties
Email Address ✓ Yes Google (Firebase)
User Content (Resources, Categories) ✓ Yes Google (Firebase)
Device ID & Advertising ID ✓ Yes Google AdMob & Ad Partners
Usage Analytics ✓ Yes Google (Firebase Analytics)
Crash Reports ✓ Yes Google (Firebase Crashlytics)
Precise Location (GPS) ✗ No Not Collected
Contacts or Phone Number ✗ No Not Collected

1. Data Collected Directly by the App

1.1 Account Information

Data Type: Email address, name, profile picture URL

Collection Method: Google Sign-In authentication

Purpose: User authentication and account management

Storage: Firebase Authentication, Cloud Firestore

Sharing: Shared with Google (Firebase) as service provider

User Control: Users can delete their account at any time

1.2 User-Generated Content

Data Type:

  • Resource titles, descriptions, and URLs
  • Category names and colors
  • Custom images uploaded by users
  • Organization preferences

Collection Method: User input through the app interface

Purpose: Core app functionality - organizing and managing resources

Storage: Cloud Firestore (structured data), Firebase Storage (images)

Sharing: Not shared with third parties (except Firebase/Google as infrastructure provider)

User Control: Users can edit or delete their content at any time

1.3 Usage Metadata

Data Type:

  • Resource access timestamps (addedAt, lastAccessed)
  • Category access timestamps
  • Device platform information (Android/iOS)

Collection Method: Automatically generated when users interact with the app

Purpose: Sorting resources by recency, improving user experience

Storage: Cloud Firestore

Sharing: Not shared with third parties

2. Data Collected by Third-Party SDKs

2.1 Firebase Services (Google)

Firebase Authentication

Data Collected: Email address, User ID (UID), authentication tokens, IP address (for security), device information

Purpose: User authentication and account security

Privacy Policy: https://firebase.google.com/support/privacy

Cloud Firestore

Data Collected: All user-generated content (resources, categories), User ID associations, timestamps

Purpose: Database storage for app functionality

Privacy Policy: https://firebase.google.com/support/privacy

Firebase Storage

Data Collected: User-uploaded images, file metadata (upload time, size), User ID associations

Purpose: Image storage for resources

Privacy Policy: https://firebase.google.com/support/privacy

Firebase Analytics

Data Collected: App usage patterns, screen views, user engagement metrics, device information (model, OS version), approximate location (country-level), app crashes and performance data

Purpose: Understanding app usage, improving features, identifying bugs

Privacy Policy: https://firebase.google.com/support/privacy

Firebase Crashlytics

Data Collected: Crash logs and stack traces, device state information, User ID (if authenticated), device model and OS version, timestamp of crashes

Purpose: Bug tracking and app stability improvement

Privacy Policy: https://firebase.google.com/support/privacy

2.2 Google Mobile Ads (AdMob)

⚠️ Important: Google AdMob collects advertising data and shares it with advertising partners for personalized ads.

Data Collected:

  • Advertising ID (AAID on Android, IDFA on iOS)
  • IP address
  • Device information (model, OS version, screen size)
  • App usage data
  • Ad interaction data (views, clicks)
  • Approximate location
  • Device identifiers

Purpose: Serving personalized advertisements, ad performance measurement

Data Sharing: Shared with Google's advertising partners and ad networks

User Control: Users can reset their advertising ID or opt out of personalized ads through device settings

Privacy Policy: https://policies.google.com/privacy

AdMob Privacy: https://support.google.com/admob/answer/6128543

How to Opt Out of Personalized Ads:

  • Android: Settings → Google → Ads → Opt out of Ads Personalization
  • iOS: Settings → Privacy → Advertising → Limit Ad Tracking

2.3 Google Sign-In

Data Collected: Google account email, profile name, profile picture, Google account ID

Purpose: Simplified user authentication

Data Sharing: Authentication tokens shared between app and Google

Privacy Policy: https://policies.google.com/privacy

3. Data Transmitted Off Device

3.1 To Firebase/Google Servers

  • All user account data
  • All user-generated content (resources, categories, images)
  • Analytics and crash data
  • Authentication tokens

Encryption: Data transmitted over HTTPS/TLS

Server Location: Google Cloud Platform data centers (region may vary)

3.2 To Google Ad Networks

  • Device identifiers and advertising IDs
  • Ad interaction data
  • Device and app usage information

Encryption: Data transmitted over HTTPS/TLS

3.3 External Links (via WebView)

When users open resource links through the app's WebView:

  • The destination website may collect standard web browsing data
  • This includes IP address, user agent, cookies, etc.
  • Note: Re-Source does not control or receive data from these external websites

URLs Accessed: Based on user-saved resources (any website URL entered by the user)

4. Data Shared with Third Parties

4.1 Direct Third-Party Sharing

  1. Google (Firebase Services): All app data as described above
  2. Google AdMob & Ad Partners: Advertising and device data as described in section 2.2

4.2 No Other Third-Party Sharing

  • We do not sell user data to third parties
  • We do not share user content with third parties (except Google as our infrastructure provider)

5. Data NOT Collected

The app does NOT collect:

  • Precise geolocation (GPS coordinates)
  • Contacts or phone numbers
  • SMS or call logs
  • Microphone or camera data (except when user chooses to upload images)
  • Calendar data
  • Photos (except when user explicitly selects for upload)
  • Health data
  • Financial information
  • Social connections outside of Google authentication

6. Data Retention

Data Type Retention Period
User Account Data Until user deletes their account
User Content (Resources, Categories) Until user deletes it or deletes their account
Analytics Data Up to 14 months (Firebase Analytics default)
Crash Logs 90 days (Firebase Crashlytics default)
Advertising Data 60-90 days (Google AdMob policy)

7. User Rights & Data Deletion

7.1 User Rights

  • Access: View all data within the app
  • Edit: Modify content at any time
  • Delete: Remove specific resources or categories
  • Account Deletion: Permanently delete account and all data
  • Data Export: Request data export by contacting the developer

7.2 Account Deletion Process

  1. Navigate to Profile → Settings in the app
  2. Select "Delete Account"
  3. Confirm deletion
  4. All user data will be permanently deleted from Firebase

Note: Anonymized analytics data may be retained for up to 14 months as per Firebase policies.

8. Data Security

Encryption in Transit

All data transmitted is encrypted using HTTPS/TLS.

  • Firebase services enforce HTTPS/TLS encryption by default
  • Google AdMob uses HTTPS/TLS for all ad requests
  • Google Sign-In uses OAuth 2.0 with encryption

Access Controls

  • Firebase Authentication security rules
  • Cloud Firestore security rules restricting data access to authenticated users only
  • Regular security updates through Firebase SDK updates

9. Children's Privacy

This app is not directed at children under 13. We do not knowingly collect personal information from children under 13. If we discover that a child under 13 has provided us with personal information, we will delete it immediately.
